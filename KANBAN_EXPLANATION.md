# KANBAN BOARD EXPLANATION

**What is a Kanban board?**  
In my own words, a Kanban board is a visual project management tool that shows the flow of work moving through different stages, like a living dashboard of everything that needs to be done.

**How our THE IT CODE ACADEMY Kanban board works**  
Our board (Sprint 1 view) is built on the Automated Kanban template and customised with additional columns to match real development needs:

- **Visualises workflow**: Columns clearly show each stage — Backlog, Ready, In Progress, Code Review, Testing, Blocked, and Done. At a glance I can see exactly where every user story from Assignment 6 is in the process.
- **Limits Work-in-Progress (WIP)**: I set a soft limit of 3 tasks in “In Progress” and “Code Review”. This stops me from starting too many things at once and helps me focus on finishing tasks, which prevents bottlenecks and keeps velocity steady.
- **Supports Agile principles**: The board makes problems visible immediately (especially in the Blocked column), encourages continuous delivery (cards move right as work progresses), and allows quick adaptation of priorities. It turns the static sprint plan from Assignment 6 into a dynamic, real-time tool that supports transparency and collaboration.

Overall, this board has already helped me stay organised and motivated while building the MVP. It’s not just a diagram — it’s a practical tool I will keep using for the rest of the project.
